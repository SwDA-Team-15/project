What is the C4 Model?

C4 provides four zoom levels to describe software architecture — from the widest view (context) down to code.

👆 YOU ARE HERE
L1

Context

The system as a black box. Who uses it? What external systems does it talk to? Great for non-technical stakeholders.

L2

Container

Zoom in to see the applications and data stores inside the system boundary — e.g. web server, database, cache.

L3

Component

Zoom into a container to see its internal components — e.g. Router, Middleware Pipeline, View Engine.

L4

Code

Classes and functions — directly maps to source files like lib/application.js.

System Under Focus
⚡

Express.js Framework

The central system. A black box at this level — we only show its interactions, not its internals. Internally it has application.js, request.js, response.js, view.js, and a Router.

External Actor
👨‍💻

Developer

The primary user of Express. They import the framework, define routes via app.get() / app.post(), chain middleware with app.use(), and start the server with app.listen().

External Actor
🌐

HTTP Client

Browsers, mobile apps, or API consumers that send HTTP requests to the running Express server and receive HTTP responses (HTML, JSON, files, redirects, etc.).

External Dependency
📦

npm Packages

22 production dependencies Express requires — notably body-parser (request parsing), router (routing engine), serve-static (static files), finalhandler (default error handling), and cookie.

Infrastructure
🟢

Node.js Runtime

Express requires Node.js ≥ 18. It uses node:http to create the HTTP server (app.listen), node:events for the EventEmitter mixin, and node:path / node:fs for view resolution and static files.

External Dependency
🖼️

Template Engines

Third-party renderers (EJS, Handlebars, Pug) registered via app.engine() and invoked through lib/view.js. Express itself is engine-agnostic; any engine matching the (path, opts, cb) contract works.

External Dependency
🔌

Middleware Ecosystem

Thousands of community npm packages mounted with app.use() — authentication (passport), logging (morgan), security (helmet), CORS (cors), sessions (express-session), and more.

Infrastructure
📡

npm Registry

The public package registry where Express is published as express@5.2.1. Developers fetch Express and all its dependencies from here. Express is funded via opencollective.com/express.

How to Create This Diagram

Three practical approaches — pick the one that fits your workflow.

APPROACH 1

Structurizr DSL (official C4 tool)

Write a .dsl file using the Structurizr language. Define person, softwareSystem, and relationship blocks, then render with structurizr-cli or the online workspace. Produces spec-compliant C4 diagrams.

APPROACH 2

Mermaid.js in Markdown

Use graph LR or the dedicated C4Context diagram type. Works in GitHub Markdown, Notion, and VS Code. Write actors as Person(), the system as System(), and relations as Rel().

APPROACH 3

Draw.io / Lucidchart

Import the free C4 shape libraries in draw.io (diagrams.net). Drag-and-drop person stickmen, system boxes, and containers. Export as PNG/SVG. Best for non-technical stakeholders who need a polished document.

APPROACH 4

PlantUML + C4-PlantUML

Include the C4Context.puml macro library, then use Person(dev,"Developer"), System(express,"Express.js"), and Rel() macros. Render locally or via plantuml.com.

KEY RULE

What belongs in Level 1?

Only show external actors (people and systems outside your boundary), the system itself as a black box, and interactions between them. No internal modules, classes, or databases at this level.

EXPRESS TIP

Identifying actors from source

Read package.json → dependencies for external deps. Check lib/application.js → app.listen() to find the runtime. Check lib/view.js for the template engine interface. Callers of createApplication() are developers.
